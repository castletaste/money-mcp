# mcp-money: Архитектурный документ

## Итоговые решения

| Аспект | Решение |
|--------|---------|
| **Позиционирование** | Трекер расходов для AI-ассистентов |
| **Объём MVP** | Транзакции + категории + теги + сводки + базовые бюджеты (16 инструментов). Без счетов. |
| **Рантайм** | Основной — Bun, совместим с Node (без Bun-специфичных API в продакшн-коде) |
| **MCP транспорт** | Только stdio через `@modelcontextprotocol/sdk` |
| **База данных** | Только PostgreSQL, через Drizzle ORM + postgres.js |
| **Схема** | 5 таблиц: categories, tags, transactions, transaction_tags, budgets. Всё в схеме `mcp_money`. |
| **Миграции** | Автомиграция при запуске с отслеживанием версий |
| **Конфигурация** | `DATABASE_URL` обязателен. `MCP_MONEY_SCHEMA` — опциональная env-переменная. Валюта по умолчанию меняется через `set_currency`, каждая транзакция хранит свою валюту. |
| **Тестирование** | Интеграционные тесты + E2E MCP-тесты через `bun test` |
| **Наблюдаемость** | Логирование в stderr + логирование SQL-запросов (DEBUG) + инструмент `health_check` |
| **Безопасность** | Изоляция через схему + параметризованные запросы |
| **Упаковка** | npm-пакет, запуск через `npx mcp-money` |
| **Open-source** | Один мейнтейнер, качество референсной реализации |
| **Запуск** | Каталоги MCP → Reddit → HN |
| **Название** | `mcp-money` |

## Схема БД

```
mcp_money.categories
  id          uuid PK
  name        text NOT NULL UNIQUE
  parent_id   uuid FK -> categories(id)  -- максимум один уровень вложенности
  type        text NOT NULL  -- 'expense', 'income'
  created_at  timestamptz

mcp_money.tags
  id          uuid PK
  name        text NOT NULL UNIQUE
  created_at  timestamptz

mcp_money.transactions
  id          uuid PK
  category_id uuid FK -> categories(id) NULLABLE
  amount      numeric(19,4) NOT NULL  -- положительное = доход, отрицательное = расход
  currency    text NOT NULL DEFAULT 'USD'
  description text
  date        timestamptz NOT NULL
  metadata    jsonb DEFAULT '{}'
  created_at  timestamptz
  updated_at  timestamptz

mcp_money.transaction_tags
  transaction_id  uuid FK -> transactions(id) ON DELETE CASCADE
  tag_id          uuid FK -> tags(id) ON DELETE CASCADE
  PRIMARY KEY (transaction_id, tag_id)

mcp_money.budgets
  id          uuid PK
  category_id uuid FK -> categories(id)
  amount      numeric(19,4) NOT NULL
  period      text NOT NULL  -- 'monthly', 'weekly'
  start_date  date NOT NULL
  created_at  timestamptz
```

**Соглашения**:
- `numeric(19,4)` для денег, никогда float
- UUID v7 для сортируемых по времени ID
- `timestamptz` для дат транзакций — хранит точное время с часовым поясом
- Отрицательные суммы = расход, положительные = доход. MCP-слой обрабатывает `add_expense(amount: 50)` → сохраняет как `-50`.
- Полная мультивалютность: каждая транзакция хранит свою валюту, сводки группируются по валюте.

## MCP-инструменты (16)

| Инструмент | Назначение |
|------------|------------|
| `add_transaction` | Записать расход или доход (принимает опциональные теги) |
| `list_transactions` | Запрос с фильтрами (диапазон дат, категория, тег, лимит) |
| `update_transaction` | Редактировать транзакцию (включая теги) |
| `delete_transaction` | Удалить транзакцию |
| `get_summary` | Расходы по категориям за период, итоги, средние |
| `list_categories` | Показать доступные категории |
| `create_category` | Добавить пользовательскую категорию |
| `delete_category` | Удалить категорию (ошибка, если есть ссылающиеся транзакции) |
| `list_tags` | Показать все теги |
| `create_tag` | Добавить новый тег |
| `delete_tag` | Удалить тег (удаляет из всех транзакций) |
| `set_budget` | Установить месячный бюджет для категории |
| `get_budget_status` | Проверить бюджет vs фактические траты |
| `delete_budget` | Удалить бюджет |
| `set_currency` | Сменить валюту по умолчанию для новых транзакций |
| `health_check` | Статус БД, версия, статистика |

**Поведение деструктивных инструментов**:
- `delete_transaction` — жёсткое удаление
- `delete_category` — RESTRICT: ошибка, если есть транзакции с этой категорией
- `delete_tag` — CASCADE: удаляет тег и все связи в transaction_tags, транзакции не затрагиваются
- `delete_budget` — жёсткое удаление

## Бизнес-логика

**Категории**: ~10 предустановленных (Groceries, Restaurants, Transport, Entertainment, Utilities, Rent, Salary, Healthcare, Shopping, Other). Пользователи добавляют свои через `create_category`. Максимум один уровень вложенности (parent_id), в v1 не форсируется.

**Теги**: Гибкие сквозные метки. Many-to-many через связующую таблицу. Создаются на лету в `add_transaction` или через `create_tag`. Без предустановок — полностью пользовательские.

**Бюджеты**: Только месячные по категории. Покрывает 90% сценариев.

**Валюта**: Полная мультивалютность. Каждая транзакция хранит свою валюту. Валюта по умолчанию (`USD`) меняется через `set_currency`, хранится в `mcp_money.settings`. Сводки группируются по валюте. Конвертация между валютами не реализуется — каждая валюта отображается отдельно.

**Миграции**: Автомиграция при запуске. Таблица версий `mcp_money.schema_version`. Применяется в транзакции, откат при ошибке. Деструктивные миграции требуют флага `--force`.

## Конфигурация

```json
{
  "mcpServers": {
    "money": {
      "command": "npx",
      "args": ["mcp-money"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost:5432/mydb"
      }
    }
  }
}
```

- `DATABASE_URL` — обязателен
- `MCP_MONEY_SCHEMA` — опционален (по умолчанию `mcp_money`)
- `DEBUG=true` — включает логирование SQL-запросов в stderr

## Журнал решений

### Отложено до пост-MVP

| Фича | Когда | Триггер |
|------|-------|---------|
| Счета | v1.1 | Запрос пользователей на разделение наличных/банк/карта |
| Повторяющиеся транзакции | v1.1 | Запрос пользователей |
| CSV импорт/экспорт | v1.1 | Очевидный ранний запрос |
| Тренды расходов | v1.2 | Когда сводки докажут полезность |
| Поддержка SQLite | v2.0 | Если Postgres блокирует adoption |
| HTTP/SSE транспорт | v2.0 | Когда MCP-клиенты массово поддержат |
| Веб-дашборд | Никогда (вероятно) | Вне скоупа MCP-сервера |

### Пересмотреть после MVP

| Решение | Пересмотреть когда |
|---------|-------------------|
| Только PostgreSQL | Если >30% issues — «как настроить Postgres» |
| Без счетов | Если >5 пользователей просят разделение транзакций по источнику |
| Конвенция знака суммы | Если пользователи/LLM путаются в отрицательных суммах |
| Автомиграция | Если хоть один пользователь сообщит о потере данных |
| Drizzle ORM | Если вызывает проблемы на Node в рантайме |
| Количество инструментов | Если LLM не справляются (уменьшить) или пользователи просят больше (расширить) |
| Только месячные бюджеты | Если >5 запросов на недельные/квартальные |
| UUID v7 | Если проблемы совместимости с драйверами |
